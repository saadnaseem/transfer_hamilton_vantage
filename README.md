# Liquid Handler Transfer File Generator

## Overview

This repository contains a Jupyter notebook (`transfer.ipynb`) that generates automated liquid handling transfer instructions. The notebook takes target media concentrations (typically from an ART optimization run) and calculates the precise transfer volumes needed to prepare media in destination plates (48-well format) using stock solutions from 24-well source plates.

### Purpose

The notebook is part of a Design-Build-Test-Learn (DBTL) cycle for media optimization. It:
1. Takes target concentrations recommended by ART (Automated Recommendation Tool)
2. Calculates transfer volumes for each component to achieve those targets
3. Determines whether to use HIGH or LOW concentration stock solutions (based on minimum transfer volume requirements)
4. Handles special components (Kanamycin, FeSO4, Culture) with custom logic
5. Generates a single CSV file with transfer instructions in a standardized format

## Output Files

### Core Output File

#### `transfer_instructions.csv`
**Location:** `data/transfer_instructions.csv` (configurable via `user_params['output_file']`)  
**Columns:**
- `Source_Plate`: Source plate identifier (e.g., 's1', 's_water')
- `Source_Well`: Source well identifier (e.g., 'A1', 'B2')
- `Dest_Plate`: Destination plate identifier (e.g., 'dest_1', 'dest_2')
- `Dest_Well`: Destination well identifier (e.g., 'A1', 'A2')
- `Transfer_Vol`: Transfer volume in µL (rounded to 2 decimal places)

**Purpose:** Complete transfer instructions for liquid handler automation. This single file contains all transfers needed to prepare all destination wells.

**Format Notes:**
- Transfers are sorted by destination plate and well
- Large transfers (>`max_tip_volume`) are automatically split into multiple records
- Water transfers come from a configurable water source (default: plate 's_water', well 'A1')
- Each component transfer is a separate row

## User Parameters

The notebook uses a `user_params` dictionary to configure all parameters:

### File Paths
- `stock_conc_file`: Path to stock concentrations CSV
- `stock_plate_high_file`: Path to high plate layout CSV
- `stock_plate_low_file`: Path to low plate layout CSV
- `target_conc_file`: Path to target concentrations CSV
- `output_file`: Path for output transfer instructions CSV

### Liquid Handling Parameters
- `well_volume`: Total volume (µL) in each destination well (default: 1500)
- `source_well_volume`: Max usable volume (µL) in each source well (default: 9000)
- `dead_volume`: Volume (µL) that cannot be aspirated (default: 100)
- `min_transfer_volume`: Minimum accurate transfer volume (µL) (default: 5.0)
- `max_tip_volume`: Maximum tip volume (µL) - transfers exceeding this will be split (default: 200.0)
- `culture_factor`: Culture dilution factor (default: 100, meaning 15µL culture in 1500µL total)
- `epsilon`: Floating point comparison tolerance (default: 1e-6)

### Plate Configuration
- `wells_per_plate`: Number of wells per destination plate (default: 48)
- `plate_format`: '48-well' or '96-well' (default: '48-well')

### Fixed Components
- `fixed_components`: Dictionary of fixed components added to all wells:
  - `MOPS`: 40.0 mM
  - `Tricine`: 4.0 mM
  - `Glucose`: 20.0 mM
  - `Kan`: 1.0 (dilution factor, where 1.0 = well_volume/300)

### Culture Handling
- `add_culture_to_all_wells`: Boolean - if True, add culture to all wells; if False, only where target_conc['Culture'] > 0

### Water Source Configuration
- `water_source`: Dictionary with:
  - `plate`: Plate name (e.g., 's_water', 'reservoir_1')
  - `well`: Well name (e.g., 'A1', or None for reservoir/trough)
  - `type`: 'plate', 'reservoir', or 'trough'

## Algorithm Overview

### Core Function: `find_volumes_bulk`

The notebook defines a `find_volumes_bulk` function that:

1. **Processes each well and component:**
   - Calculates transfer volume: `transfer_vol = (target_conc × well_volume) / stock_conc`
   - Selects stock level (high/low) based on minimum transfer volume requirements
   - Tries HIGH stock first; switches to LOW if volume < `min_transfer_volume`

2. **Special component handling:**
   - **Kanamycin (Kan)**: dont worry about it 
   - **FeSO4**: dont worry about it 
   - **Culture**: Uses volume ratio (not concentration): `transfer_vol = well_volume / culture_factor`

3. **Water calculation:**
   - Calculated as: `water_vol = well_volume - sum(all_component_volumes)`
   - Validated to ensure non-negative (give warning if negative)

4. **Returns:**
   - `df_volumes`: DataFrame (Well × Component) with transfer volumes
   - `df_conc_level`: DataFrame (Well × Component) with stock level ('high', 'low', 'fresh', or None)

5. the capacity of stock concentration well is given. if after calculating the volume, if the total volume of a component exceeds than assign another/extra well to that component in stock concentration plate 
### Transfer Record Building

After volume calculations:

1. **Source well assignment:**
   - Uses `stock_lookup` dictionary (created from stock plate layouts) to map components to source wells
   - Handles duplicate components by tracking cumulative volume usage
   - Checks for source well depletion (warns if total usage > usable volume)

2. **Transfer splitting:**
   - If `transfer_vol > max_tip_volume`, splits into multiple records
   - Number of cycles: `ceil(transfer_vol / max_tip_volume)`
   - Volume per cycle: `transfer_vol / num_cycles`

3. **Destination plate assignment:**
   - Assigns wells to destination plates based on `wells_per_plate`
   - Supports well remapping for different plate formats (48-well vs 96-well)

   - Each split transfer becomes a separate row in the output

6. **Source Well Depletion**
   - Notebook tracks cumulative volume usage per source well
   - Warns if total usage exceeds `source_well_volume - dead_volume`

9. **Water Source Configuration**
   - Water source is configurable (plate, well, type)
   - Supports unlimited sources (reservoir/trough) which don't track depletion
   - Default: plate 's_water', well 'A1', type 'plate'

## Validation and Testing

The notebook includes comprehensive validation:

### Volume Sum Validation
- Checks that total volume per well equals `well_volume`
- Uses epsilon tolerance for floating point comparison

### Minimum Transfer Volume Validation
- Warns if any transfer volume < `min_transfer_volume`
- Notes which stock level was used

### Water Volume Validation
- Ensures water volumes are non-negative
- Warns if water volume is very small (< `min_transfer_volume`)

### Back-calculation Validation
- Validates that calculated concentrations match target concentrations
- Formula: `calculated_conc = (transfer_vol × stock_conc) / well_volume`
- Reports mean and max absolute errors

### Source Depletion Checks
- Tracks cumulative volume usage per source well
- Warns if usage exceeds usable volume (`source_well_volume - dead_volume`)
